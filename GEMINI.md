# G-ESG (Green ESG Platform) 업무 수행 지침

본 문서는 G-ESG 프로젝트 내에서 모든 작업을 수행할 때 반드시 준수해야 하는 기초 지침이다.

## 1. 공통 원칙
- **언어**: 모든 답변과 코드 주석은 **한글**로 작성한다.
- **코드 무결성**: 기존 보안 및 디자인 기능을 절대 삭제하거나 생략하지 않는다.

## 2. 보안 및 액세스 제어 (CRITICAL)
- **로그인 시스템 보존**: 
    - 페이지 접속 시 반드시 로그인 오버레이가 먼저 표시되어야 한다.
    - 계정 정보: 이름(`sayuri`) / 암호(`!seoyoung12`) 로직을 절대 수정하거나 삭제하지 않는다.
    - 인증 성공 전에는 메인 콘텐츠(`main-content`)가 노출되지 않도록 엄격히 관리한다.

## 3. UI/UX 디자인 가이드
- **에스테틱**: 다크 모드 및 글래스모피즘(Glassmorphism) 스타일을 유지한다.
- **메인 메시지**: 로그인 성공 후 표시되는 "Hello, ESG !!!" 문구와 스타일을 보존한다.
- **반응형 디자인**: 모바일 및 다양한 해상도에서 로그인 카드가 중앙에 위치하도록 유지한다.

---
**위 지침은 사용자의 명시적인 수정 지시가 없는 한 영구적으로 유효하며, 모든 코드 생성 시 최우선 순위로 고려한다.**
